通过wix4制作golang程序的windows安装包

Go WixToolset
技术
发布日期:   2024-05-15
文章字数:   2.8k

以系统服务运行程序

在windows上运行golang程序的时候总会有一个令人讨厌的运行窗口，我们可以使用第三方库来将程序以windows系统服务的方式运行。
使用库：https://github.com/kardianos/service
直接来看库里的简单示例：https://github.com/kardianos/service/blob/master/example/simple/main.go

// simple does nothing except block while running the service.
package main

import (
	"log"

	"github.com/kardianos/service"
)

var logger service.Logger

type program struct{}

func (p *program) Start(s service.Service) error {
	// Start should not block. Do the actual work async.
	go p.run()
	return nil
}
func (p *program) run() {
	// Do work here
}
func (p *program) Stop(s service.Service) error {
	// Stop should not block. Return with a few seconds.
	return nil
}

func main() {
	svcConfig := &service.Config{
		Name:        "GoServiceExampleSimple",
		DisplayName: "Go Service Example",
		Description: "This is an example Go service.",
	}

	prg := &program{}
	s, err := service.New(prg, svcConfig)
	if err != nil {
		log.Fatal(err)
	}
	logger, err = s.Logger(nil)
	if err != nil {
		log.Fatal(err)
	}
	err = s.Run()
	if err != nil {
		logger.Error(err)
	}
}

在上述代码中，我们首先创建了一个program结构体，并实现了Service接口的Start和Stop方法。
在Start方法中，我们启动了一个协程来执行实际的服务逻辑。
run方法就是你自己的服务逻辑代码，你可以在其中编写你需要的功能。
然后，我们创建了一个Config对象来配置服务的名称、显示名称和描述。
接下来，我们创建了一个Service对象并使用New函数初始化。
最后，我们调用Run方法来以服务方式运行我们的程序。

在命令行中，你可以使用以下命令将Go程序编译为Windows服务：

go build -o main.exe

main.exe是生成的服务可执行文件的名称。
然后，你可以使用以下命令来安装服务：

main.exe install

安装成功后，你可以使用以下命令来启动、停止或卸载服务：

main.exe start
main.exe stop
main.exe uninstall

但是实际使用中会有点不同，我们只需要在windows下进行服务化就行了，mac和linux本身就有服务化运行的方式，不用多次一举，所以需要判定下系统环境：

func main() {
	runtime.GOMAXPROCS(runtime.NumCPU())

	prg := &program{}
	if runtime.GOOS == "windows" {
		svcConfig := &service.Config{
			Name:        "User",
			DisplayName: "UserService",
			Description: "User Service.",
		}
		s, err := service.New(prg, svcConfig)
		if err != nil {
			println(err)
			return
		}
		serviceLogger, err = s.Logger(make(chan error, 5))
		if err != nil {
			println(err)
			return
		}
		if len(os.Args) > 1 {
			err = service.Control(s, os.Args[1])
			if err != nil {
				serviceLogger.Errorf("Failed (%s) : %s\n", os.Args[1], err.Error())
				return
			}
			serviceLogger.Errorf("Succeeded (%s)\n", os.Args[1])
			return
		}
		err = s.Run()
		if err != nil {
			serviceLogger.Errorf("Run Failed : %s\n", err.Error())
		}
	} else {
		prg.run()
	}
}

类似于这样，不是windows的话，直接运行主流程逻辑prg.run()就行。

安装wix4

wix最新的是版本5，其中4和5都是比较新的，3版本就比较老了，差异很大，不能直接拿来用，如果你们有老项目是用wix3打包的话，有个简单的方法可以将wix3的配置文件一键转换成wix4。
使用Visual Studio：

直接在右侧的解决方案管理器中右键项目就能看到“升级到wix4”的选项。

这是后续话题了，我们应该先安装好wix4，直接看官网文档：https://wixtoolset.org/docs/intro/

是说wix4已经加入了NET工具，使用NET指令安装即可，安装完打印下版本号检查下是否安装成功。

然后在windows上下载 visual studio 2022 ，这个IDE完全支持wix的编辑和运行。
首先打开空白的vs，点击顶部选项栏里的“扩展”->“管理扩展”，在“联机”一栏里搜索“wix”，安装wix插件：

HeatWave是制作WiX脚本的辅助工具。

制作配置文件

刚开始的时候全程使用vs来编辑wix配置文件，可以便修改便验证，主要是方便，因为制作过程中是不可能一帆风顺的。

创建新项目，选择MSI模板，创建完成就会进入编辑界面，左边是文件编辑界面，右边是解决方案管理界面，这里就不放了

然后在解决方案管理界面，右键“依赖项”->“管理NuGet程序包”，安装必要的插件，安装好之后，配置文件自动会同步更新，这里安装下面三个，选择对应的4版本

  • WixToolset.Heat
  • WixToolset.UI.wixext
  • WixToolset.Util.wixext

然后就是根据项目需求进行对应的配置编辑了。
这里我直接放示例，然后再挑重点说明一下。

userService.wixproj

<Project Sdk="WixToolset.Sdk/4.0.5">
    <ItemGroup>
        <PackageReference Include="WixToolset.Heat" Version="4.0.5" />
        <PackageReference Include="WixToolset.UI.wixext" Version="4.0.5" />
        <PackageReference Include="WixToolset.Util.wixext" Version="4.0.5" />
    </ItemGroup>

    <PropertyGroup>
        <DefineConstants>BasePath=..\build\win</DefineConstants>
    </PropertyGroup>

    <PropertyGroup Condition="'$(Configuration)|$(Platform)'=='Debug|x86'">
        <Cultures>
        </Cultures>
    </PropertyGroup>

    <ItemGroup>
        <Content Include="filter.xslt" />
        <Content Include="assets/UserService.ico" />
    </ItemGroup>

    <ItemGroup>
        <HarvestDirectory Include="..\build\win">
            <ComponentGroupName>PublishedComponents</ComponentGroupName>
            <DirectoryRefId>INSTALLFOLDER</DirectoryRefId>
            <SuppressRootDirectory>true</SuppressRootDirectory>
            <KeepEmptyDirectories>false</KeepEmptyDirectories>
            <SuppressRegistry>true</SuppressRegistry>
            <PreprocessorVariable>var.BasePath</PreprocessorVariable>
            <SuppressCom>true</SuppressCom>
            <Transforms>filter.xslt</Transforms>
        </HarvestDirectory>
        <BindPath Include="..\build\win" />
    </ItemGroup>
</Project>

这个是项目主配置文件，其文件名决定了最终生成的安装包的名称，比如 userService.wixproj 最终生成的msi安装包是 userService.msi

这里比较重要的是这一串：

<ItemGroup>
    <HarvestDirectory Include="..\build\win">
        <ComponentGroupName>PublishedComponents</ComponentGroupName>
        <DirectoryRefId>INSTALLFOLDER</DirectoryRefId>
        <SuppressRootDirectory>true</SuppressRootDirectory>
        <KeepEmptyDirectories>false</KeepEmptyDirectories>
        <SuppressRegistry>true</SuppressRegistry>
        <PreprocessorVariable>var.BasePath</PreprocessorVariable>
        <SuppressCom>true</SuppressCom>
        <Transforms>filter.xslt</Transforms>
    </HarvestDirectory>
    <BindPath Include="..\build\win" />
</ItemGroup>

作用是将项目中“**..\build\win**”下的内容打包进msi包，比如前端静态文件，将其打包进去，安装后才会重新生成，否则没有前端文件，将无法访问前端界面。

package.wxs

这个是安装包配置文件，配置安装包安装过程中的UI、逻辑等，由于需要着重说明的比较多，我直接在代码上注释了

<Wix xmlns="http://wixtoolset.org/schemas/v4/wxs">

  <!-- 定义常量 -->
  <?define UpgradeCode = "bc1adac3-47ed-4374-9c21-1298243b6ed7" ?>
  <?define BuildVersion = "1.0.0.0" ?>

  <!-- 可以定义安装界面显示的名称、语言、版本号等属性 -->
  <Package Name="!(loc.ProductName)" Language="!(loc.Language)" Manufacturer="!(loc.CompanyName)" Version="$(var.BuildVersion)" UpgradeCode="$(var.UpgradeCode)" Codepage="65001">

    <!-- 定义安装包摘要信息（Summary Information）的一部分 -->
    <SummaryInformation Manufacturer="!(loc.CompanyName)" Description="!(loc.Description)" Keywords="!(loc.Keywords)" />

    <!-- 这行代码表示在执行主要升级时，如果发生了降级（即安装了旧版本），则显示从本地化字符串文件中读取的降级错误消息 -->
    <MajorUpgrade DowngradeErrorMessage="!(loc.DowngradeError)" />

    <!-- 定义升级策略的一部分。它指定了在安装新版本时如何处理旧版本的升级 -->
    <!-- Id 属性用于指定升级代码 -->
    <Upgrade Id="$(var.UpgradeCode)">()
        <!-- 指定特定版本的升级策略。在这里，它指定了升级的最低版本和最高版本 -->
        <!-- OnlyDetect 属性指示是否仅检测旧版本而不执行实际升级。在这里，它设置为 no，表示执行实际升级 -->
        <!-- Property 属性指定了一个用于存储正在升级的旧版本号的属性名称 -->
        <UpgradeVersion Minimum="0.0.0.1" IncludeMinimum="yes" OnlyDetect="no" Maximum="$(var.BuildVersion)" IncludeMaximum="no" Property="OLDERVERSIONBEINGUPGRADED" />
    </Upgrade>

    <!-- 定义媒体模板的一部分。媒体模板用于指定安装包在创建安装媒体（如安装 CD、DVD 或安装程序下载文件）时的行为 -->
    <!-- EmbedCab 属性用于指定是否在安装媒体中嵌入 CAB 文件。CAB 文件是一种用于存储 Windows 安装程序文件的压缩文件格式。在这里，设置为 "yes" 表示应该将 CAB 文件嵌入到安装媒体中 -->
    <MediaTemplate EmbedCab="yes" />

    <Icon Id="icon.ico" SourceFile="assets/userService.ico" />

    <Property Id="ARPPRODUCTICON" Value="icon.ico" />

    <!-- 定义用户界面（UI） -->
	<UI Id="UserInterface">
	    <Property Id="WIXUI_INSTALLDIR" Value="TARGETDIR" />

		<TextStyle Id="WixUI_Font_Normal" FaceName="Tahoma" Size="8" />
		<TextStyle Id="WixUI_Font_Bigger" FaceName="Tahoma" Size="12" />
		<TextStyle Id="WixUI_Font_Title" FaceName="Tahoma" Size="9" Bold="yes" Red="255" Green="255" Blue="255" />

		<Property Id="DefaultUIFont" Value="WixUI_Font_Normal" />
		<Property Id="WixUI_Mode" Value="Custom" />

		<DialogRef Id="ProgressDlg" />
		<DialogRef Id="ErrorDlg" />
		<DialogRef Id="FilesInUse" />
		<DialogRef Id="FatalError" />
		<DialogRef Id="UserExit" />

		<Publish Dialog="ExitDialog" Control="Finish" Event="EndDialog" Value="Return" Order="999" />
		<Publish Dialog="WelcomeDlg" Control="Next" Event="EndDialog" Value="Return" Order="2"></Publish>
	</UI>

    <!-- 引用了一个名为 "WixUI_Common" 的预定义用户界面 -->
    <!-- 在使用 <UIRef> 元素引用预定义用户界面时，WiX 工具集会自动将相应的对话框和行为添加到您的安装程序项目中 -->
	<UIRef Id="WixUI_Common" />

    <!-- 配置安装界面的两张制定位置的图片 -->
    <WixVariable Id="WixUIDialogBmp" Value="assets/Background.bmp" />
    <WixVariable Id="WixUIBannerBmp" Value="assets/Banner.bmp" />

    <!-- 这里引入两个组件，在下一个文件中 -->
    <Feature Id="Main">
      <ComponentGroupRef Id="DataFolderFiles" />
      <ComponentGroupRef Id="PublishedComponents" />
    </Feature>
  </Package>
</Wix>

folders.wxs

这个文件则配置要运行的程序，也就是我们编译好的go运行程序

<Wix xmlns="http://wixtoolset.org/schemas/v4/wxs">
  <Fragment>
	  <DirectoryRef Id="INSTALLFOLDER">
		  <Component Id="UserServiceEXE" Guid="{8B834EB9-4597-4629-BA5C-2C964F3E261C}">
			  <File Id="UserService" Source="..\build\win\userService.exe" KeyPath="yes" />
			  <ServiceInstall Id="UserService" Type="ownProcess" Vital="yes" Name="UserService" DisplayName="UserService" Description="UserService" Start="auto" Account=".\LocalSystem" Interactive="no" ErrorControl="ignore" />
			  <ServiceControl Id="StartUserService" Name="UserService" Start="install" Wait="no" Stop="both" Remove="uninstall" />
			  <RemoveFile Id="RemoveUserServiceFiles" Name="userService.exe" On="uninstall" />
			  <Environment Id="GIN_MODE" Name="GIN_MODE" Value="release" Permanent="no" Part="all" Action="set" System="yes" />
		  </Component>
	  </DirectoryRef>

	  <ComponentGroup Id="DataFolderFiles">
		  <ComponentRef Id="UserServiceEXE" />
	  </ComponentGroup>
	  
    <StandardDirectory Id="ProgramFiles6432Folder">
      <Directory Id="INSTALLFOLDER" Name="!(bind.Property.ProductName)" />
    </StandardDirectory>
  </Fragment>
</Wix>

语言文件

另外还可以配置多语言，项目中有几个语言文件，wix对应的会打包这些语言的安装包，比如下面的是英文语言包 en-us.wxl：

<WixLocalization xmlns="http://wixtoolset.org/schemas/v4/wxl" Culture="en-US">
    <String Id="DowngradeError" Value="A newer version of [ProductName] is already installed." />
    <String Id="ProductName" Value="UserService" />
    <String Id="ProductNameFolder" Value="UserService" />
    <String Id="Language" Value="1033" />
    <String Id="CompanyName" Value="Ali Technology Pte. Ltd." />
    <String Id="Description" Value="User Service for Windows" />
    <String Id="Keywords" Value="dotnet, core, application, installer, User Service" />
</WixLocalization>

对应的日文的文件则叫：ja-jp.wxl，以此类推

自动化安装脚本

以上，我们制作好了wix打包配置文件，将它们放入项目目录中，这是目录结构：

userService
-……
-angular
-golang
-build_win.bat
-win.install
–assets
–userService.wixproj
–package.wxs
–folders.wxs
–en-us.wxl

这是自动化打包脚本，包括前后端项目的编译等：

@echo off
echo %1

set BUILDNUMBER=%1
echo %BUILDNUMBER%

@REM set INCLUDE_PATH=%PROJECT_ROOT%\GoLangDockerImage\include
@REM set LIBRARY_PATH=%PROJECT_ROOT%\Libs\windows

set CGO_ENABLED=1
@REM set CGO_CFLAGS=-I%INCLUDE_PATH%
@REM set CGO_CXXFLAGS=-I%INCLUDE_PATH%
@REM set CGO_LDFLAGS=-L%LIBRARY_PATH%

cd .\angular

echo -- install packages --
call npm install --legacy-peer-deps

echo -- display ng version --
call ng version
echo -- start build angular --
call ng build --build-optimizer

cd ..
mkdir build
mkdir .\build\win
cd .\golang
echo -- create win printer code --
cd .\controller

echo -- build Coordinator generator --
cd ..
go mod tidy
set GOOS=windows
set GOARCH=amd64
go build -ldflags "-X main.UserServiceVersion=%BUILDNUMBER%" -v -o ..\build\win\userService.exe main.go

cd ..\
mkdir .\build\win\Public
xcopy /s /y .\angular\dist\* .\build\win\Public

echo -- generate and run .msi --
cd .\Kepler.Install
dotnet build --force -c Release

@REM cd .\bin\Release\en-US
@REM sc delete UserService
@REM timeout /t 15
@REM msiexec /i UserService.msi

完成，执行bat脚本就能自动打包成msi安装包了
